THE APP

A full-stack e-commerce application built for learning. Products include the "Git Commit Hoodie", "404 Not Found Mug", "Rubber Duck Debugger", and more.

01

Backend

Java 17 + Spring Boot 3.2

  • H2 in-memory database with JPA/Hibernate
  • REST controllers, services, repositories
  • Global exception handling
  • Data seeder for product catalog
02

Frontend

React 18 + TypeScript + Vite

  • React Router for navigation
  • CartContext for global state
  • Axios API client
  • Plain CSS with BEM naming
03

E2E Testing

Playwright

  • Page Object Model pattern
  • Product, Cart, Checkout tests
  • data-testid selectors

AI Fundamentals & Prompt Engineering Deep Dive →

Prompting Techniques

  • Zero-shot — Direct instruction, no examples
  • Few-shot — Provide input/output examples to guide the model
  • Chain-of-thought — "Think step by step" for complex reasoning
  • Role prompting — "You are a senior Java developer..."

Effective Prompts for Code

  • Be specific about language, framework, and patterns
  • Reference existing code with @ mentions
  • Include constraints ("under 150 lines", "no external deps")
  • Provide context about the project architecture
  • Ask for tests alongside implementation

SESSION 3 — 9:15 - 10:30

MCP, Skills, Commands & Subagents Deep Dive →

MCP Architecture

  • Model Context Protocol — Standard for connecting AI to external tools
  • Configured in .cursor/mcp.json
  • Filesystem, GitHub, and custom MCP servers
  • JSON-based configuration with dynamic loading

Custom Commands

  • Plain markdown files in .cursor/commands/
  • Invoked with /name in agent chat
  • Single-action prompt shortcuts
  • Examples: /review, /add-endpoint

Skills

  • Multi-step workflows in .cursor/skills/<name>/SKILL.md
  • YAML frontmatter with description and name
  • Can include supporting scripts and templates
  • Invoked with /skill-name or @skill-name
  • Create interactively with /create-skill

Subagents

  • Specialized agents in .cursor/agents/
  • YAML frontmatter: name, description, model, readonly, is_background
  • Auto-delegated based on description or explicit /name
  • Built-in: Explore, Bash, Browser
  • Background agents use git worktrees for isolation

SESSION 5 — 1:00 - 2:15

AI Governance & Hooks Deep Dive →

Code Review for AI-Generated Code

  • SQL injection, XSS, and OWASP Top 10 checks
  • Authentication and authorization gaps
  • Anti-patterns: god classes, over-engineering, copy-paste
  • BugBot automated PR review and "Fix in Cursor"

Project Rules

  • .cursor/rules/*.mdc — MDC format with frontmatter
  • Always Apply — alwaysApply: true
  • Glob-based — globs: ["backend/**/*.java"]
  • Intelligent — Agent decides from description
  • Manual — User @-mentions the rule
  • Create interactively with /create-rule

Agent Hooks

  • Configured in .cursor/hooks.json
  • Scripts that run before/after agent actions
  • afterFileEdit — Auto-format code
  • beforeShellExecution — Block dangerous commands
  • beforeReadFile — Redact secrets from LLM
  • sessionStart/stop — Audit logging
  • Response: allow, deny, or ask

Hook Configuration

{
  "version": 1,
  "hooks": {
    "afterFileEdit": [{
      "command": ".cursor/hooks/format.sh",
      "timeout": 10,
      "failClosed": false
    }]
  }
}

REFERENCE

The .cursor/ Ecosystem

Every primitive is version-controlled and shared with your team. Here's how they fit together.

📋

Rules

Always-on coding standards

Persistent instructions injected into every AI interaction. Define coding conventions, architectural patterns, and project-specific constraints that the agent follows automatically.

When to use "Always use constructor injection in Spring Boot." "React components must be functional with TypeScript." "Never use var in Java." Any standard the team should follow on every prompt, without remembering to ask.
.cursor/rules/*.mdc
⚡

Commands

Quick prompt shortcuts

Single-action prompts triggered by typing /name in chat. Think of them as saved prompts — one markdown file, one action, no supporting files needed.

When to use "Review this file for security issues." "Generate a unit test for the selected code." "Explain this error." Any repeatable, single-step AI action you find yourself typing over and over.
.cursor/commands/*.md
🧠

Skills

Multi-step workflows

Structured, multi-step instructions that guide the agent through a complete workflow. Can include supporting files like templates, scripts, and examples in a dedicated folder.

When to use "Scaffold a full-stack feature (model → repo → service → controller → React component)." "Set up a new microservice with all boilerplate." Any multi-step recipe that needs to be done the same way every time.
.cursor/skills/*/SKILL.md
🤖

Subagents

Parallel specialist agents

Custom agent personas with specific capabilities and constraints. Can be read-only, run in the background on separate git worktrees, and use faster models for efficiency.

When to use "Run all tests and report back while I keep coding." "Verify the build in the background." "Explore the codebase and summarize the architecture." Any task you want to delegate to a specialist that works independently or in parallel.
.cursor/agents/*.md
🚧

Hooks

Lifecycle guardrails

Shell scripts that execute before or after agent actions. They can allow, deny, or prompt for confirmation — giving you programmatic control over what the agent does.

When to use "Auto-format every file the agent edits." "Block rm -rf and git push --force." "Log every agent session for compliance." "Redact secrets before the LLM reads a file." Any guardrail that must be enforced, not just suggested.
.cursor/hooks.json
🔌

MCP Servers

External tool connections

Model Context Protocol servers that give the agent access to external tools and data sources — databases, APIs, browsers, file systems, design tools — through a standardized interface.

When to use "Query the Postgres database directly." "Browse the running app with Playwright." "Pull components from a Figma design." "Read and create GitHub issues." Any time the agent needs to interact with something outside the code editor.
.cursor/mcp.json
📦

Plugins

Bundled distribution

Packages that bundle rules, skills, commands, and MCP servers into a single installable unit. Distributed through the Cursor Marketplace or shared as URLs.

When to use "Install Stripe's official Cursor plugin for payment integration." "Share our team's entire .cursor/ setup as one package." "Distribute a partner SDK integration." Any time you want a one-click setup of pre-configured AI tooling.
Marketplace

When to Use What

I want to... Use Location
Enforce coding standards automatically Rule .cursor/rules/*.mdc
Run a quick AI-driven action Command .cursor/commands/*.md
Define a multi-step workflow Skill .cursor/skills/<name>/SKILL.md
Delegate specialized work in parallel Subagent .cursor/agents/*.md
Run scripts before/after agent actions Hook .cursor/hooks.json
Connect to external tools & data MCP Server .cursor/mcp.json
Bundle everything for distribution Plugin Cursor Marketplace

Commands vs Skills vs Rules

Aspect Commands Skills Rules
Invocation On-demand /name On-demand /name or @name Automatic
Scope Single action Multi-step workflow Coding standards
Format Plain markdown SKILL.md + frontmatter .mdc + frontmatter
Supporting files No Yes (scripts, templates) No
Best for "Do this one thing" "Here's how to do X" "Always follow these"
